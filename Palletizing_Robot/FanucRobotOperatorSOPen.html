Safety

Do not enter robot operation zone while robot is on.

Whenever entering the cell, remove the safety key and carry it on your person. Do not return the safety key until you have vacated the cell. Assure that all personnel are outside of the robot cell before activating the robot by visually inspecting the cell. If a person is inside the robot cell with the robot operating, press the E-stop immediately.

Pressing the E-stop causes a power stop. This locks up the Servo motors and causes the robot to stop instantly. The E-stops are located on each side of the robot.

E-stop location

Breaking the gate barrier causes an emergency stop. When this happens, the robot will apply the motor breaks at their maximum load, which is not good for the robots internal mechanism.

For entering the cell in a non-emergency, always proper procedure for stopping the robot.

Overhead objects: Watch out for robot overhead when in cell

Falling objects: Be aware of dropped boxes

Electrical: High voltage used

Tripping hazard: Cable guards are tripping hazard

Fork lifts and industrial vehicles: Be aware of moving vehicles around robot

Beacon Lights

There are four beacon lights on the robot. Each has a different meaning.

The red light indicates that the robot has a fault. The robot has stopped and needs human intervention to continue operation.

 

The yellow light indicates that a stop request has been made.

 

 

The green light indicates normal operation.

 

 

The blue light indicates that a pallet is full and ready to be removed

 

 

 

 

Standard Operation

Indexing a Pallet

The robot runs autonomously and only requires human interaction when a pallet is full or a fault occurs.

When a pallet is fully loaded, the indicator light shines blue as shown.

Indicator lights

If you are already familiar with standard pallet replacement procedures, continue as you have done.

 

Standard pallet replacement procedure is as follows:

1)      Press the yellow button on the side of the gate.

2)      Wait for the indicator light to turn red.

3)      Remove Key

The robot cell is now safe to enter.

4)      Remove full pallet. Using Toyota pallet mover, remove the pallet from the robot cell.

5)      Place new pallet. In pallet slot. Make sure that the pallet is pushed in flush so that the sensors recognize that a pallet is inserted.

6)      Exit cell and return Key. Turn key and make sure the indicator light turns off.

7)      Press the blue button to reset the fault. This will take a moment.

8)      Press the green button to start the robot

The robot is now operational again.

 

Clearing a Buffer

Sometimes, such as at the end of an order, it is necessary to physically clear a buffer of the robot. Here is the procedure to do this.

1)      Press the yellow button to stop the robot and remove the safety key.

2)      Physically remove the cases from the buffer. Place them outside the robot cell.

3)      Exit the robot cell and return the safety key, but do not start the robot.

4)      In the HMI, navigate to the buffers screen

5)      Then, to clear a buffer in the HMI press both [clr] buttons for that buffer.

7)      Press the blue button and then the green button to start the robot.

Dropped Box

A dropped box triggers the robot to have a vacuum sensor fault. Identify if the box has been dropped or if there is a false alarm. If it has been dropped, press (2) on the HMI. If it is a false alarm, press (1).

When the box has been dropped, collect and remove the remnants of the box and any scattered bags. Place these bags into a fresh box.

Make sure to place the correct label on the new box. Then, place these boxes back onto the conveyor. You may choose to seal the box or not. If you choose to seal place the case after the case sealer, here:

If unsealed, place before the prepresser so that it may be pressed and sealed. Adjust the number of cases in the HMI to the correct number of boxes on each pallet (page 13). DO NOT LET THE CASES TOUCH

 

Smashed Box

A smashed box is caused by the robot thinking that there is not a box in a location where this is one. There are always at least 2 smashed boxes: the box the robot is holding and the box(s) that are being smashed.

Remove the damaged boxes. Place the bags from these boxes into fresh, undamaged boxes.

Make sure to place the correct label on the new box. Then, place these boxes back onto the conveyor. You may choose to seal the box or not. If you choose to seal place the case after the case sealer, here:

If unsealed, place before the prepresser so that it may be pressed and sealed. Adjust the number of cases in the HMI to the correct number of boxes on each pallet (page 13). DO NOT LET THE CASES TOUCH

 

A-B-A Error

This error is caused by a glitch in the program under VERY specific circumstances. To describe this we will refer to Case A as cases that contain bag type A, and Case B as cases that contain bag type B. Case A and Case B are both labeled to go to the same pallet. Here is what must happen for this error to occur:

(1)    The robot receives case type A and places this case on a buffer.

(2)    The pallet case that A and B are assigned to is indexed (removed) and a fresh pallet is put in.

(3)    The robot receives case type B and places it on the new pallet.

(4)    The robot receives case type A again.

This causes the robot to fail to run task because it cannot reconcile the buffer case with the pallet case. The best method for addressing this issue is prevention.

PREVENTION: To prevent this A-B-A Error, when a pallet is indexed for being the last of an order, check to make sure that there are no cases of the same type (ID) as the removed pallet in any of the buffers. If there are cases of the same type (ID) in a buffer, remove the case, and clear the number of buffered cases in the HMI (page 13).

OCCURANCE: If prevention has failed, the solution is much more involved but is very similar to the procedure for adjusting the number of cases. It is recommended that you bring the supervisor for assistance.

Press (MENU) and navigate to the [DATA] and select [String Registers]

Use the arrow keys to navigate down to SR[28: Pal8 ID ON_PAL ]

SR[21 to SR[28 tell you the ID on all the pallets

Compare these case ID numbers with the case ID numbers of recently removed pallet. Going further down on the DATA String Registers page will bring you to the buffers:

When a buffer has cases, it will show the ID of the cases instead of empty.

Determine which case type is erroneous- case A or case B. It may help to check cases on the conveyor and to ask your supervisor.

Empty the buffer(s) that contains case type A. Remove the pallet that has case B.

Barcode Read Error

This error occurs when the barcode cannot be read. Follow proper procedure to stop the robot and enter the cell. Check that the boxs orientation is correct and assure that the box is labeled. If one or both of these are not done, correctly orient and label the case. DO NOT PUSH THE CASE BACK. DO NOT LET THE CASES TOUCH. Once this has been done, place the corrected box back on the conveyor and start the robot.

Incorrect Index Time

On rare occasion the robot will indicate that a pallet is ready to be indexed when it is not. Here is how to proceed when this incorrect indexing occurs.

1)      Press the yellow button to stop the robot.

2)      Remove the pallet that is falsely indexed from its location.

3)      Place the pallet back into the palletizing bracket. This clears the index error but also resets the case count to 0.

4)      Exit the cell. Do not start the robot yet.

5)      Abort the program in the HMI.

6)      Press the blue button and then the green button.

7)      Follow the prompted procedures. Remove the first box from the pickup zone.

8)      Adjust the number of boxes on the pallet to the corrected number and layer.

9)      Press the blue button and then the green button to start the robot.

Buffer Mismatch Error

Here are the steps to take if there is a Buffer Mismatch error. This error can be noted by the flashing red light instead of the typical solid red for an error.

1)      Press the yellow button to stop the robot.

2)      Check the HMI buffers screen. Next to one of the buffers should be a red dot indicating an error.

3)      Empty the cases from the buffer that has the red dot. Make sure to also clear the case numbers (both) from the HMI.

4)      Abort the program in the HMI.

5)      Press the blue and then the green button.

6)      Follow the prompted procedures on the HMI.

7)      Adjust the number of boxes to the correct number and layer if needed.

8)      Press done on the HMI.

9)      Wait while the robot places a few boxes. The robot may throw the error again, and if this happens repeat steps 1-8. This time it will show a different buffer with the red dot.

10)   If the problem persists, then the incoming boxes are problematic. Check the labels and make sure that the pallet letter is what it should be.


 

MAIN Page Disappeared

This can occur by accident when trying to resolve a different error. Typically to access the MAIN page you press the (MENU) button then tap [MAIN] on the screen. If after pressing (MENU), the [MAIN] option does not appear on the screen, follow these steps.

1)      Press the function (FCTN) button

2)      Press the left arrow on the keypad

3)      Press (1) to select QUICK/FULL MENUS

4)      Press the (MENU) button

5)      Tap [MAIN] on the screen

Robot Wont Place Box on Pallet

1)      Make sure the robot is otherwise running as normal

2)      Check that the label on the box is printed correctly and that the QR code is reading correctly

3)      Check the keys on all the safety gates. Make sure that none of the red LED lights are on.

4)      Check that the pallet is fully inserted into the yellow barrier.

5)      Check that the sensors on both the left and right side of the pallet are reading close to 0. If any is reading above 50, adjust the pallet position
